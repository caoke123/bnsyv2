# Design Reference Library 分析报告

> 来源：[nexu-io/open-design](https://github.com/nexu-io/open-design)
> 分析日期：2026-06-18
> 分析范围：linear-app / raycast / warp / vercel / supabase

---

## 1. Linear App

### 特点

Linear 是**暗色模式原生的极致精确设计系统**。它以近乎纯黑的画布 (`#08090a`) 为基底，内容以星光般的方式从黑暗中浮现。这不是一个「为浅色主题套上暗色皮肤」的设计——黑暗本身就是原生媒介。

- 字体系统完全基于 **Inter Variable**，全局启用 OpenType 特性 `cv01` (单层 a) + `ss03` (几何变体)，赋予字体更干净、更具几何感的特征
- **510 字重** 是 Linear 的标志性字重——介于 Regular (400) 和 Medium (500) 之间，创造出微妙而有力的强调感
- 颜色几乎完全是消色的——仅靠一个品牌紫罗兰色 (`#5e6ad2` / `#7170ff`) 点缀 CTA、激活态和品牌元素
- 边框系统使用半透明白色 (`rgba(255,255,255,0.05)` ~ `0.08`)，如同月光下绘制的线框
- 组件基于 **Radix UI** 原语

### 布局特点

- 8px 基础网格，主节奏 8/16/24/32px
- 最大内容宽度约 1200px
- Hero 居中单列 + 宽裕的垂直内边距
- 特性区域：2-3 列卡片网格
- 全宽暗色区段，内部有最大宽度约束
- 区段之间 80px+ 垂直间距，无可见分割线——暗色背景提供自然分隔

### 卡片设计

- 背景始终为半透明：`rgba(255,255,255,0.02)` ~ `0.05`（从不使用实色）
- 边框：`1px solid rgba(255,255,255,0.08)`（标准）或 `0.05`（微妙）
- 圆角：8px（标准）/ 12px（特色）/ 22px（大面板）
- hover 时背景透明度微妙提升
- 多层阴影叠加，包含 inset 变体

### 导航设计

- 暗色固定顶栏，位于近乎纯黑的背景上
- Linear logo 左对齐
- 链接：Inter Variable 13-14px, 字重 510, 颜色 `#d0d6e0`
- 激活态/hover：文字亮至 `#f7f8f8`
- CTA：品牌靛蓝按钮或幽灵按钮
- 移动端：汉堡菜单折叠
- 支持命令面板 (`Cmd+K`)

### 状态设计

- 成功绿 (`#27a644`, `#10b981`) 仅用于状态指示器
- 深度层级系统：Flat → Subtle → Surface → Inset → Ring → Elevated → Dialog (5 层)
- 焦点态：`rgba(0,0,0,0.1) 0px 4px 12px` 多层叠加
- 阴影哲学：在暗色表面上，传统阴影不可见，改用边框-即-阴影技术 + 多层叠加

### 日志设计

- 无 Changlog 系统（此为产品设计系统，非日志系统）

### 适用场景

- 项目管理工具 / SaaS 产品
- 需要极高精度和工程感的开发者工具
- 暗色模式为主的复杂数据仪表盘
- 追求「少即是多」的品牌体验

## 2. Raycast

### 特点

Raycast 的设计感如同**精密仪器的暗色内部——用黑曜石雕刻的瑞士表壳**。背景是微妙的蓝调暗色 (`#07080a`)，营造出嵌入 macOS 原生应用的沉浸感，而非普通网页。

- 签名动作是**借自 macOS 窗口 chrome 的分层阴影系统**：带 inset 高亮的多层 box-shadow，模拟物理深度
- **Raycast Red** (`#FF6363`) 几乎只用于 Hero 的标志性斜线条纹——红色是标点符号，不是弥漫色
- Inter 字体配合正面字母间距 (+0.2px ~ +0.4px)，在暗色界面上营造出异常通透的呼吸感
- 键盘快捷键的键帽样式：渐变背景 + 多层阴影模拟真实物理按键
- 组件基于 Radix UI 原语

### 布局特点

- 8px 基础网格，区段间距 80px-120px
- 最大宽度约 1200px（断点 1204px）
- Hero 单列，特性区域 2-3 列网格
- 产品截图以 macOS 窗口 chrome 风格居中展示
- 区段在广袤的暗色虚空中漂浮，营造电影般的节奏感

### 卡片设计

- 表面色 `#101111`，边框 `1px solid rgba(255,255,255,0.06)`
- 圆角 12px-16px
- **双环包容**：外环 `rgb(27,28,30) 0px 0px 0px 1px` + 内环 `rgb(7,8,10) 0px 0px 0px 1px inset`
- 微妙暖光 (`rgba(215,201,175,0.05)`) 在特色元素背后营造氛围
- hover 时通过边框透明度变化微妙增亮

### 导航设计

- 暗色顶栏与页面背景融为一体
- 链接：Inter 16px 字重 500，默认 `#9c9c9d`，hover 变白
- CTA 按钮：半透明白色药丸形 (86px 圆角)
- 移动端汉堡菜单折叠
- 固定定位 + 微妙底部边框分隔

### 状态设计

- 所有按钮 hover 使用**透明度变化** (opacity 0.6) 而非颜色变化——这是 Raycast 的标志性交互模式
- 错误/警告/成功使用对应色彩强调：Red / Yellow / Green，均带半透明辉光
- 4 层键盘键帽阴影模拟物理按压效果
- 5 层深度：Void → Subtle → Ring → Button(macOS 原生玻璃按钮) → Key(物理 3D) → Floating(命令面板)

### 日志设计

- 无日志系统

### 适用场景

- macOS 原生感的产品启动器 / 效率工具
- 需要精密、工具感界面的开发者产品
- 键盘快捷键密集的交互类应用
- 需要营造「高性能桌面工具」信任感的产品

## 3. Warp

### 特点

Warp 的设计是**开发者工具中最具人性的品牌表达**——如同一台终端坐在篝火旁的深林里：温暖、暗沉、安静自信。与大多数开发者工具偏爱的冷调蓝色黑不同，Warp 将一切包裹在暖色近黑中，感觉像烧焦的木头或深色泥土。

- 文本不是纯白，而是**暖羊皮纸色** (`#faf9f6`)，微妙奶油感柔化每个标题
- 字体使用 **Matter**——几何无衬线体，与众不同地具有柔和、人性化的特质，传达「这个终端是为每个人设计的」
- 几乎单色的暖灰调色板——**没有大胆的强调色**
- 在终端截图之间穿插**自然摄影**，创造「生活方式 × 开发者工具」的独特叙事
- 大写标签配宽字母间距 (1.4px-2.4px)，营造杂志编辑式分类系统
- 按钮为深色药丸形 (`#353534`, 50px 圆角)，克制、低调

### 布局特点

- 8px 基础网格
- 最大宽度约 1500px（断点 1500px）——宽幅电影式布局
- 全宽 Hero，双列特性区段（摄影 + 文本并排）
- 单列评价展示
- 宽阔的容器让摄影呼吸
- 编辑式节奏感：每个区段刻意的翻页时刻

### 卡片设计

- 自然摄影卡片：全出血图片 + 叠加文本，圆角 8px-12px
- 边框卡片：半透明边框 (`rgba(226,226,226,0.35)`) 提供包容，圆角 12px-14px
- hover 几乎无变化——保持静谧美学
- 终端截图卡片：产品 UI 嵌入暗色容器

### 导航设计

- 暗色顶栏，Matter Regular 16px 链接
- 链接颜色：`#868584`（石灰色），激活/hover 变为暖羊皮纸色
- CTA：深色药丸 (`#353534`)，低调不抢眼
- 移动端折叠简化导航
- 固定定位

### 状态设计

- 近乎零阴影——深度通过半透明边框、摄影层次和表面不透明度变化传达
- 没有玻璃态或模糊效果
- 交互状态通过下划线装饰和不透明度变化传达
- 没有颜色化的焦点环——与单色调色板保持一致
- 任何暖光都来自摄影的自然光照，而非人工 CSS

### 日志设计

- 无日志系统

### 适用场景

- 面向非纯技术用户的开发者工具（降低门槛）
- 需要传递温暖、人性化品牌调性的技术产品
- 生活方式 × 工具结合的营销站点
- 避免「冷硬技术感」的同时保持专业感

## 4. Vercel

### 特点

Vercel 的设计是**开发者基础设施隐形的视觉论文**——一个如此克制的设计系统，近乎哲学宣言。页面压倒性白色 (`#ffffff`)，配合近黑文字 (`#171717`)，创造出画廊般的空旷感，每个元素都必须争取自己的像素。这不是作为装饰的极简主义，而是作为工程原则的极简主义。

- 自定义 **Geist** 字体是皇冠上的明珠——在展示尺寸下使用 -2.4px 到 -2.88px 的极端负字母间距，产生被压缩、被 minify 的代码感
- **阴影即边框**哲学：`box-shadow: 0px 0px 0px 1px rgba(0,0,0,0.08)` 替代传统 CSS border，使边框存在于阴影层，实现更平滑的过渡和更轻的视觉重量
- 全局启用 OpenType `"liga"`（连字）
- 工作流特定强调色：Ship Red (`#ff5b4f`)、Preview Pink (`#de1d8d`)、Develop Blue (`#0a72ef`)
- Geist Mono 作为代码/技术标签的等宽伴侣

### 布局特点

- 8px 基础网格，存在 16px→32px 的显著跳跃（无 20px/24px）
- 最大内容宽度约 1200px
- Hero 居中单列 + 宽裕顶部内边距
- 特性区域：2-3 列卡片网格
- 全宽分隔使用 `border-bottom: 1px solid #171717`
- 120px+ 垂直间距——**白色空间就是设计本身**

### 卡片设计

- 纯白背景 (`#ffffff`)，阴影提供边框和深度
- 完整卡片阴影栈：`rgba(0,0,0,0.08) 0px 0px 0px 1px`(边框) + `rgba(0,0,0,0.04) 0px 2px 2px`(提升) + `rgba(0,0,0,0.04) 0px 8px 8px -8px`(环境) + `#fafafa 0px 0px 0px 1px`(内部微光)
- 圆角：8px（标准）/ 12px（特色/图片卡片顶部）
- 图片卡片额外 `1px solid #ebebeb`
- hover 时阴影微妙增强

### 导航设计

- 干净的白色水平导航，固定定位
- Vercel 标志左对齐 (262×52px)
- 链接：Geist 14px 字重 500，`#171717`
- 激活态：字重 600 或下划线
- CTA：深色药丸按钮（"Start Deploying", "Contact Sales"）
- 移动端汉堡菜单折叠
- 产品下拉菜单支持多级

### 状态设计

- 焦点环：`2px solid hsla(212, 100%, 48%, 1)`——饱和蓝，对无障碍至关重要的统一焦点指示
- 几乎无颜色变化——深度完全来自阴影层次和边框对比
- 没有背景色变化区分区段
- 5 层深度：Flat → Ring → Light Ring → Subtle Card → Full Card → Focus

### 日志设计

- 无日志系统

### 适用场景

- 前端部署 / 开发者基础设施产品
- 需要传达「工程级精确」和「零冗余」的品牌
- 白色为主的 SaaS 营销站点
- 追求画廊级品质感的极简主义设计
- 需要复杂阴影系统展示深度的产品

## 5. Supabase

### 特点

Supabase 的设计是**暗色模式原生的开发者平台**，汇聚了高级代码编辑器的美学——深黑背景 (`#0f0f0f`, `#171717`) 搭配翡翠绿强调色 (`#3ecf8e`, `#00c573`)，引用了品牌的开源、PostgreSQL 绿色基因。

- 字体使用 **Circular**——带有圆润终端的几何无衬线体，软化技术感的边缘
- 72px Hero 文本 line-height 1.00——文本被压缩到绝对零行距，像终端命令一样高效
- 等宽伴侣 **Source Code Pro** 用于大写技术标签 (1.2px 字母间距)
- **HSL 颜色令牌系统**配合 alpha 通道，实现半透明层次化着色
- 绿色强调色极为克制——仅限 logo、链接和强调边框 (`rgba(62,207,142,0.3)`)
- 药丸按钮 (9999px) 仅用于主要 CTA，与标准 6px 次要按钮形成清晰层次

### 布局特点

- 8px 基础网格，大间距跳跃: 48px → 90px → 96px → 128px
- 居中内容，宽裕最大宽度
- 全宽暗色区段 + 内部内容约束
- 特性区域：图标网格 + 统一卡片
- 显著简单的断点系统——主要仅一个 600px 断点，暗示移动优先+渐进增强
- 电影式区段间隔 (90px-128px)，每个区段在暗色虚空中自成一幕

### 卡片设计

- 暗色表面 (`#171717` 或稍亮)
- 边框：`1px solid #2e2e2e` 或 `#363636`
- 圆角 8px-16px
- **无可见阴影**——边框定义边缘
- 内部 padding 16px-24px
- 深度通过**边框阶层**定义：`#242424`(几乎不可见) → `#2e2e2e`(标准) → `#363636`(突出)

### 导航设计

- 暗色背景与页面一致 (#171717)
- Supabase logo + 绿色图标
- Circular 14px 字重 500
- 干净的水平布局 + 产品下拉菜单
- 绿色 "Start your project" 药丸 CTA
- 固定 header

### 状态设计

- **刻意回避阴影**——在暗色模式原生设计中，传统阴影几乎不可见且毫无意义
- 深度通过边界颜色差异传达：`#242424` → `#2e2e2e` → `#363636` → 绿色强调边框
- 焦点态仅使用 `rgba(0,0,0,0.1) 0px 4px 12px`——极简功能主义
- 绿色强调边框 (`rgba(62,207,142,0.3)`) 作为「已提升」状态——品牌色本身成为深度信号
- 无玻璃态或模糊效果
- Radix 色彩语义令牌 (crimson, purple, violet, yellow) 用于各种状态

### 日志设计

- 无日志系统

### 适用场景

- 开源数据库 / BaaS 平台
- 需要开发者信任感的技术产品
- 暗色模式为主的后台管理系统
- 需要简洁、高效、零装饰的工具型产品

## 7. Recommended Design Mix

针对网点系统自动化项目的后台管理系统，推荐以下设计混合方案：

### 基础骨架：Supabase (60%)

- **暗色模式原生**：`#171717` 页面背景，适合长时间操作的运营后台
- **边框阶层系统**：用 `#242424` → `#2e2e2e` → `#363636` 替代阴影，简洁高效
- **HSL Alpha 层次化**：用半透明实现复杂的层次感而不增加视觉负担
- **简洁布局**：单断点 (600px) 哲学，移动优先

### 精确感注入：Linear (20%)

- **510 字重**：介于 Regular 和 Medium 之间的微妙强调，塑造独特的 UI 语气
- **半透明白色边框**：`rgba(255,255,255,0.05)` ~ `0.08`，月光线框感
- **品牌色克制使用**：仅 CTA、激活态、状态指示器
- **命令面板 (Cmd+K)**：高效导航的交互模式

### 动画与深度：Raycast (10%)

- **按钮 hover 透明度过渡**：opacity 变化而非颜色变化，流畅自然
- **Inset 阴影**：模拟 macOS 原生玻璃按压效果
- **双环包容技术**：外环 + 内环 inset 形成物理感

### 排版精度：Vercel (10%)

- **负字母间距**：标题使用紧凑间距，营造工程感
- **OpenType 连字**：启用 `liga`，提升代码和技术文本的排版质量

### 不建议引入的元素

- Warp 的自然摄影融合风格（与后台管理系统场景不符）
- Vercel 的纯白背景（后台系统不适合大面积白色）
- Raycast 的键盘按键 3D（后台系统不是键盘驱动型应用）

### 设计原则总结

1. **黑暗是原生媒介**——不为浅色设计「套上」暗色皮肤
2. **深度来自边框，而非阴影**——在暗色背景上阴影几乎不可见
3. **强调色是标点符号，不是弥漫色**——单色系统中的稀缺色彩才有力量
4. **排版即身份**——字体选择、字重、间距比颜色更能传达品牌性格
5. **少即是工程**——每个元素都必须挣得它的像素
