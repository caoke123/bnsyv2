## 二、待跟进（功能未完成 / 存在缺陷 / 阻塞项）

### 2.1 高优先级

| # | 问题 | 位置 | 影响 | 紧急度 |
|---|------|------|------|--------|
| 1 | **无超时控制、无取消机制** | `AssignmentEngine.ts:102-111` | Handler挂起→任务+锁永久持有 | **高** |
| 2 | **allResults 内存无界增长 + 全量 stringify O(n²) 写放大** | `AssignmentEngine.ts:77,86` | 大任务(万单级)可能OOM/CPU飙升 | **高** |
| 3 | **僵尸任务不清理** - `cleanupRunningTasks` 空实现 | `routes.ts:408-410` | 重启后 running 任务永久僵尸，状态不准 | **高** |
| 4 | **SIGINT 不等待在途任务** | `index.ts:121-127` | 服务停止丢失进行中的任务数据 | **高** |
| 5 | **无 checkpoint 续跑** | 全模块缺失 | 任务中断后必须从头开始，不可恢复 | **高** |
| 6 | **非原子写入** - 解析失败静默清空 `result_data` | `AssignmentEngine.ts:86` | 一次JSON序列化失败→全部结果永久丢失 | **高** |
| 7 | **openBrower 重复调用 + 无并发限制** | `BrowserPool.ts` | EasyBR启动配额快速耗尽 | **高** |
| 8 | **锁无 TTL** - 超时检测形同虚设 | `WindowLockManager.ts` | 锁泄漏→窗口永久不可用 | **高** |

## 六、风险与建议

### 6.1 当前最大风险点

| 等级 | 风险 | 一句话描述 |
|------|------|-----------|
| **P0** | 无超时/无取消/无心跳 | 任务一旦启动即"黑盒"，卡死则锁永久持有 |
| **P0** | 无恢复/无续跑/无重试 | 失败后人工干预成本极高，重启后running任务永久僵尸 |
| **P0** | 内存无界 + 全量stringify | 大任务可能OOM+CPU飙升 |
| **P0** | 非原子写入 | 单次JSON序列化失败→全部结果永久丢失 |
| **P0** | EasyBR无熔断/无并发限制 | EasyBR启动配额快速耗尽，服务不可用时请求堆积 |
| **P1** | 到件扫描静默标记成功 | 业务数据准确性风险(查询/分页失败→空表格提交) |
| **P1** | 进度回调并发竞态 | 多员工并发→result_data互相覆盖 |
| **P1** | 前端死代码 + 页面重复 | 维护成本高，新人理解困难 |
| **P2** | 测试覆盖率不足 | Handler/Engine/SessionManager 均无测试 |
